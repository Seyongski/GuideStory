# PLANS.md — 개발 단계 로드맵 & 백로그

> 개발 프로세스는 4개 단계(1 → 2 → 2.5 → 3)로 진행한다.
> 각 단계는 [ARCHITECTURE.md](ARCHITECTURE.md)의 ADR 및 증명 과제와 연결된다.

## 1. 단계별 마일스톤

### 1단계 — 로컬 2D 플랫포머 (기반)
- [ ] 타일맵 배치 및 캐릭터 이동
- [ ] 발판 착지 및 점프 물리 구현
- [ ] (선행) `Vector2D`/`Rect` 자체 정의, `RenderDevice`/`Window` 인터페이스 — ADR-006
- [ ] (선행) `TextureManager`/`WindowManager` RAII + Custom Deleter — ADR-002
- 연결: ADR-002, ADR-006

### 2단계 — 데이터 및 콘텐츠 확장
- [ ] FSM 기반 몬스터 배치
- [ ] AABB 충돌 체크
- [ ] CSV/JSON 스킬·아이템 데이터 로드 시스템 (`DataManager` + 파싱 예외 처리)
- [ ] QuadTree 충돌 최적화 + 브루트포스 vs 쿼드트리 벤치마크
- [ ] (전환) 상속 → 컴포넌트 기반 게임 오브젝트
- 연결: ADR-003, ADR-004, ADR-005

### 2.5단계 — C++ 엔진과 AI 서브시스템 통합
- [ ] Python 학습 환경 구성 (ML 에코시스템)
- [ ] C++ ↔ Python 통신 레이어 (ZMQ 또는 Socket)
- [ ] libtorch로 학습된 모델을 C++ 엔진에서 직접 로드·추론
- 연결: ADR-007

### 3단계 — 멀티플레이어 풀스택 전환
- [ ] 클라이언트 코드를 서버/클라 구조로 분리
- [ ] WinSock 기반 패킷 구조체 정의
- [ ] 멀티스레드 서버에서 여러 플레이어가 한 맵에서 동기화되는지 검증
- 연결: ADR-001

## 5. 작성 규칙

- 각 항목은 **검증 가능한 완료 조건**을 가진다.
- 증명 과제가 걸린 항목은 **코드 + 분석 문서**가 모두 끝나야 완료.
- 완료 시 백로그 → 완료 로그로 이동하고 [ARCHITECTURE.md](ARCHITECTURE.md) 증명 대시보드 상태를 갱신한다.
