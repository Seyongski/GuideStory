# docs/game-design.md — 게임 디자인 바이블

> GuideStory의 게임플레이 단일 진실 원천(SSOT).
> 본 프로젝트는 **엔진/구조 증명**이 1차 목적이므로, 게임 디자인은 그 기술을 시연하기 위한 최소 콘텐츠를 지향한다.

## 1. 장르 & 레퍼런스

- 장르: **2D 사이드스크롤링 플랫포머 RPG** (메이플스토리류)
- 레퍼런스: 메이플스토리 (타겟 직무: 메이플스토리 콘텐츠 프로그래밍)

## 2. 핵심 판타지

> 플레이어는 사이드스크롤 맵을 누비며 점프 액션과 스킬로 몬스터를 처치하는 **캐릭터**가 된다.

## 3. 핵심 루프

```
이동/점프(플랫포밍) → 몬스터 조우 → 스킬 사용 → 처치/드롭 → 진행
```
- 모든 행동은 클라이언트 입력 → **서버 판정** → 브로드캐스트로 처리(멀티플레이 단계). 1단계는 로컬.

## 4. 디자인 기둥

1. **기술 시연 우선** — 콘텐츠는 엔진/구조를 증명하기 위한 도구.
2. **데이터 주도** — 스킬·몬스터·아이템 수치는 코드가 아닌 외부 데이터.
3. **서버 권위** — 게임플레이의 진실은 서버에 있다.

## 5. 게임플레이 요소

### 5.1 플레이어
- 이동(좌/우), 점프, 발판 착지 물리.
- 스킬 사용 (데이터 정의: 쿨타임, 타격 횟수, 범위, 데미지 계수, 이펙트 프레임).
- 스킬 상태이상 등은 **컴포넌트**로 확장. (→ [../ARCHITECTURE.md](../ARCHITECTURE.md) ADR-003)

### 5.2 몬스터
- **FSM 기반** 행동 (예: Idle → Patrol → Chase → Attack → Dead).
- 스펙(HP, 공격력, 이동속도 등)은 외부 데이터 파일로 정의.
- (선택) 2.5단계에서 일부 행동을 학습된 AI 추론으로 대체/보강.

### 5.3 충돌 & 전투
- AABB 기반 충돌.
- 대량 몬스터/드롭 시 QuadTree로 후보 탐색. (→ ADR-004)

### 5.4 아이템 / 드롭
- 몬스터 처치 시 드롭. 아이템 정의는 외부 데이터.

## 6. 콘텐츠 데이터 스키마 (개요)

| 콘텐츠 | 주요 필드 | 포맷 | 관할 |
|--------|-----------|------|------|
| 스킬 | id, 쿨타임, 타격횟수, 범위, 데미지계수, 이펙트프레임 | JSON/CSV | DataManager |
| 몬스터 | id, HP, 공격력, 이동속도, FSM 파라미터 | JSON/CSV | DataManager |
| 아이템 | id, 종류, 효과, 드롭확률 | JSON/CSV | DataManager |

(상세 필드/예시는 [product-specs.md](product-specs.md) 및 [mechanics-harness.md](mechanics-harness.md)에서 확정)
