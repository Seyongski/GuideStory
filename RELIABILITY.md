# RELIABILITY.md — 안정성 & 테스트 정책

## 1. 목표

GuideStory가 의도한 시나리오를 **크래시·행·데이터 손상 없이** 완주하도록 보장한다.

## 2. 오류 처리 원칙

- 복구 불가능한 상태는 **빠르게 실패**시키고 명확한 메시지를 남긴다.
- 외부 입력(JSON/CSV 데이터, 세이브, 설정, 에셋, **네트워크 패킷**)은 신뢰하지 않고 검증한다.
- 리소스(SDL 핸들/메모리/파일/소켓)는 **RAII + 스마트 포인터(Custom Deleter)** 로 관리한다. (→ ADR-002)
- 널/경계/오버플로를 의식해 작성한다. (C++20)

### 2.1 데이터 파싱 (ADR-005)
- `DataManager`의 JSON/CSV 파싱은 **데이터 오염을 가정**한다: 누락 필드·형식 오류·범위 초과를 검출.
- 오류 시 크래시 대신 **명확한 진단 + 안전 폴백 또는 로드 중단**. 부분 오염이 게임 상태로 새지 않게 한다.

### 2.2 네트워크 / 멀티스레드 (ADR-001)
- 서버는 권위자: 클라이언트 패킷을 검증 후 적용. 비정상 패킷은 거부/로깅.
- 공유 게임 상태는 동기화로 보호. 데이터 레이스/데드락을 의식해 락 범위를 최소화.
